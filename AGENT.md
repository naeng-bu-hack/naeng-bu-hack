# AGENT Guide (for Codex)

## Scope
- 요청한 범위만 변경하고, 추가 확장은 사전 합의 후 진행한다.
- 아키텍처/의존성 변경이 필요하면 먼저 이유와 영향을 짧게 공유한다.

## Implementation
- 작은 단위로 구현하고, 각 단계에서 실행 가능한 상태를 유지한다.
- 설정값은 하드코딩하지 않고 `.env` 기반으로 읽는다.
- 공용 로직(설정/유틸)은 파일로 분리하고 재사용한다.

## Safety
- 파괴적 명령(`rm -rf`, `git reset --hard`)은 사용자 명시 요청 없이는 실행하지 않는다.
- 기존 변경사항을 임의로 되돌리지 않는다.
- 민감정보는 커밋하지 않는다(`.env` 커밋 금지, `env.example`만 관리).

## Validation
- 수정 후 최소 1회 실행 검증을 한다.
- 가능하면 API 기본 경로(`/health`, `/docs`) 확인 후 결과를 공유한다.
- 실행 불가 환경이면, 막힌 원인과 재현 커맨드를 함께 남긴다.

## Communication
- 무엇을/왜 바꿨는지 파일 단위로 명확하게 보고한다.
- 불확실한 요구사항은 추정하지 말고 질문으로 확정한다.
- "다음 액션"을 1~2개로 제안해 의사결정을 빠르게 한다.
